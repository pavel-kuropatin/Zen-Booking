  Букинг:

- Зарегистрированный пользователь может снимать жильё и/или сдавать его.
- Съём доступен сразу после регистрации.
- Клиенты могут снимать апартаменты на определённое число дней, бронировать апартаменты,
  а также отказываться от брони, если дата съёма не наступила.
- История заказов каждого клиента сохраняется и доступна им для просмотра.
- Для съёма жилья необходимо выбрать предложение собственности, выбрать доступную дату, оставить заказ и ждать пока хост примет его.
- Хост может отклонить заказ.
- После совершения заказа можно оставить отзыв.
- Отзывы отображаются на странице собственности с указанием даты бронирования и сортируются по orders.start_date
- ПЛАНИРУЕТСЯ: К отзыву можно приктеплять фотографии.

- Для сдачи жилья нужно добавить его на свой аккаунт и добавить минимум одно изображение.
- Можно добавить фотографии жилья или изменить их в любое время.
- Жильё и фотографии должны быть одобрены модератором перед добавлением в листинг.
- В любое время владелец может убрать собственность из листинга без удаления и вернуть обратно.
- При получении заказа владелец может принять его или отклонить.

  Администрирование:

- Роль ADMIN существует изначально.
- Роль MODER может создать и банить только ADMIN.
- Модераторы должны подтверждать собственность и картинки, банить и прочее.
- ПЛАНИРУЕТСЯ: Администратор может простатривать действия модераторов.
- ПЛАНИРУЕТСЯ: Модераторы могут просматривать историю своих действий.

  Эндпоинты для юзеров:

  /user/                                                  - просмотр всех юзеров/добавление юзера
  /user/{userId}/                                         - просмотр/изменение/удаление юзера {userId}

  /user/{userId}/property/                                - просмотр всей собственности/добавление собственности юзера {userId}
  /user/{userId}/property/{propertyId}/                   - просмотр/изменение/удаление собственности {propertyId} юзера {userId}

  /user/{userId}/property/{propertyId}/image/             - просмотр всех изображений/добавление изображения собственности {propertyId} юзера {userId}
  /user/{userId}/property/{propertyId}/image/{imageId}/   - просмотр/удаление изображения {imageId} собственности {propertyId} юзера {userId}

  /user/{userId}/property/review/                         - просмотр всех отзывов юзера {userId}
  /user/{userId}/property/{propertyId}/review/{reviewId}/ - просмотр/изменение/удаление отзыва {reviewId} юзера {userId}

  /user/{userId}/order/                                   - просмотр всех заказов юзера {userId}
  /user/{userId}/order/{orderId}/                         - просмотр/изменение/удаление заказа {orderId} юзера {userId}

  Под вопросом:

  /property/                                              - просмотр всей собственности, доступной для заказа юзера, поиск по ней
  /property/{propertyId}/                                 - просмотр собственности {propertyId} и её заказ
  /property/{propertyId}/review/                          - просмотр всех отзывов собственности {propertyId}
  /property/{propertyId}/review/{reviewId}/               - просмотр отзыва {reviewId} собственности {propertyId}
